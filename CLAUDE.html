Build

This is a static academic website generated by Webify, a custom Python tool (written by Faisal) that wraps Pandoc to process Markdown, YAML, and Mustache/Jinja2 templates into HTML. The generated HTML lands in ~/Sites/faisalqureshi.github.io and is pushed to GitHub Pages separately from this repo.

./mkweb.sh                  # build site to ~/Sites/faisalqureshi.github.io
./mkweb.sh --some-flag      # pass extra flags to webify

Output goes to ~/Sites/faisalqureshi.github.io. The .webifyignore file lists paths excluded from the build (shell scripts, templates directory, dot files, etc.).

Architecture

The site is data-driven: YAML files hold structured data, Markdown files hold prose content, and Mustache/Jinja2 templates render HTML.

Key data files (YAML): - research.yaml - Research projects, datasets, code, and nostalgia items shown on the homepage - students.yaml - Current and alumni student profiles (photo, name, role, links) - teaching.yaml - Current and past courses - news.yaml - News items - datasets.yaml - Dataset catalog - code.yaml - Open source code projects - _partials/nav.yaml - Navigation structure (links and nested submenus)

Templates live in _templates/: - index.jinja2 - Homepage - project.mustache - Individual research project pages - students.jinja2 - Student gallery page - dataset.mustache - Dataset pages - Various reusable partial templates (img.mustache, img-grid.mustache, vid.mustache, etc.)

Content directories: - projects/ - One subdirectory per research project, each with a Markdown file and images - pubs/ - Publication PDFs - datasets/ - Hyperspectral dataset documentation and files - _partials/ - Shared HTML fragments (header, footer, nav)

Content conventions

Each content Markdown file has YAML front matter that specifies its renderer and template:

---
render: "./_templates/project.mustache"
preprocess-mustache: True
html-img: "./_templates/img.mustache"
renderer: mustache
---

. resolves to the template root at build time. Use triple braces `` for unescaped HTML in Mustache templates.

Image sizes: - Student profile photos: 64x64 px (referenced in students.yaml) - Project thumbnails: 128x128 px (referenced in research.yaml) - Animated GIFs: convert -delay 100 -loop 0 -resize 128x128 *.jpg animated.gif

Adding a new student: add an entry to students.yaml and place a 64x64 photo in imgs/students/.

Adding a new project: create a subdirectory under projects/, add a Markdown file with the front matter above, add images, and add an entry (with 128x128 thumbnail) to research.yaml.
